# Architecture

## Scope

This repository implements a learning-focused ecommerce customer-service backend. It supports order fact lookup, policy retrieval, confirmable after-sales drafts, handoff drafts, conversation-scoped image evidence, and auditable execution records. Images stay in local conversation asset storage and are not indexed into Milvus.

## Agent Responsibilities

| Component | Responsibility | Writes business state |
| --- | --- | --- |
| `RouterAgent` | Classifies intent, entities, and risk without side effects. | No |
| `SupervisorAgent` | Chooses specialist order and synthesizes guarded replies. | No |
| `ProductAgent` | Reads demo product data. | No |
| `OrderAgent` | Reads customer-owned order facts. | No |
| `KnowledgeAgent` | Retrieves Markdown policy knowledge with citations. | No |
| `VisionAgent` | Extracts structured evidence from the current uploaded image. | No |
| `AfterSalesAgent` | Creates a pending after-sales draft. | Draft only |
| `HandoffAgent` | Creates a pending handoff draft. | Draft only |

## Workflow

```text
Input -> RouterAgent -> SupervisorAgent -> Specialist execution
      -> ResponseGuard -> Supervisor synthesis
      -> [draft action?] interrupt confirmation
      -> authorized submission or cancellation -> ResponseGuard
```

Image requests add the visual step before the normal workflow:

```text
multipart image -> LocalAssetStorage(conversation scope)
                -> VisionAgent structured evidence
                -> usable evidence: existing after-sales flow
                -> uncertain evidence: existing handoff draft flow
                -> interrupt confirmation
```

`AgentRuntime` uses LangGraph checkpointing for confirmation pauses. `AuthorizedToolExecutor` and SQLite enforce customer ownership, action status transitions, and turn leases so a ticket is created only through the confirmation path.

## RAG

```text
Markdown files
  -> MarkdownHeaderTextSplitter
  -> sentence nodes + neighbor window metadata
  -> local dense embedding + Milvus built-in BM25
  -> metadata allow-list filter + RRF
  -> KnowledgeAgent answer + context citations
```

Evaluation results are generated by `python-impl/scripts/evaluate_rag.py`.

## Audit Boundary

`AgentRun` persists each conversation workflow call with the invoked agents, run status, pending action id, reply, and timestamp. `ToolCall` records authorized tool calls. The project still does not connect to a real external order system, real refund processor, or browser UI.
